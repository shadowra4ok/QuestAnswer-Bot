# 🤖 QA Bot — Генератор ответов на типовые вопросы

Telegram-бот, который автоматически отвечает на частые вопросы студентов в чате группы. Использует **TF-IDF + косинусное сходство + лемматизацию** для понимания перефразированных вопросов без словаря синонимов.

## Как это работает

### Архитектура

```
Студент → Telegram → bot.py → QAEngine → Ответ
                                  │
                        ┌─────────┴─────────┐
                        │   1. Лемматизация  │  pymorphy2
                        │   2. TF-IDF        │  scikit-learn
                        │   3. Cosine Sim    │  numpy
                        │   4. Порог → ответ │
                        └────────────────────┘
```

### Пайплайн обработки вопроса

1. **Лемматизация** — слова приводятся к начальной форме (`сдавать` → `сдавать`, `курсовую` → `курсовой`), удаляются стоп-слова (`когда`, `можно`, `а`).
2. **TF-IDF векторизация** — каждый вопрос превращается в числовой вектор, где значение каждого слова = его важность.
3. **Косинусное сходство** — вычисляется угол между вектором пользователя и каждым вектором базы.
4. **Порог уверенности** — если лучший score < 0.15, бот отказывается отвечать вместо того, чтобы давать неверный ответ.

## Математика

### TF-IDF (Term Frequency — Inverse Document Frequency)

Формула для слова *t* в документе *d* из корпуса *D*:

```
TF-IDF(t, d, D) = TF(t, d) × IDF(t, D)
```

Где:
- **TF(t, d)** = (число вхождений t в d) / (общее число слов в d)
- **IDF(t, D)** = log(|D| / |{d ∈ D : t ∈ d}|)

Смысл: слово важно, если оно часто встречается в конкретном вопросе (TF высокий), но редко — во всех вопросах базы (IDF высокий). Слово «экзамен» встречается во многих вопросах → низкий IDF → менее полезно для различения. Слово «шпаргалка» встречается только в одном → высокий IDF → отлично различает.

### Косинусное сходство

```
cos(θ) = (A · B) / (||A|| × ||B||)
```

Где A и B — TF-IDF векторы двух вопросов. Результат ∈ [0, 1]: 0 = ничего общего, 1 = идентичны.

Геометрически: измеряем угол между векторами в многомерном пространстве слов. Чем меньше угол — тем ближе смысл.

### Почему это работает без словаря синонимов

Пример: `«срок сдачи работы»` vs `«когда сдавать курсовую»`.

После лемматизации:
- Запрос: `срок сдача работа`
- База: `сдавать курсовой работа`

Общее слово `работа` + `сдача`/`сдавать` (одна лемма!) дают ненулевое сходство. TF-IDF усиливает вес редких слов, отсекая шум.

## Быстрый старт

### 1. Получить токен бота

Откройте [@BotFather](https://t.me/BotFather) в Telegram → `/newbot` → скопируйте токен.

### 2. Локальный запуск

```bash
git clone <repo-url> && cd qa-bot
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
export BOT_TOKEN="ваш_токен"
export ENGINE_TYPE="tfidf_synonyms"  # опционально, это дефолт
python bot.py
```

#### Доступные движки (`ENGINE_TYPE`)

| Значение           | Описание                                              |
|--------------------|-------------------------------------------------------|
| `tfidf`            | Базовый TF-IDF + лемматизация                        |
| `tfidf_synonyms`   | **Дефолт.** Добавляет замену синонимов и сленга      |
| `embeddings`       | Char n-grams (3–5) + word bigrams, два векторизатора  |

### 3. Запуск через Docker

```bash
cp .env.example .env
# Отредактируйте .env — вставьте токен
docker compose up -d --build
```

### 4. Сравнение движков

```bash
python eval_compare.py
```

Выведет accuracy, precision, recall, F1 для всех трёх движков на 33 тестах. Сохранит графики в `results/`.

## Индикация уверенности

Бот показывает уровень уверенности в ответе:

- 🟢 **> 50%** — высокая уверенность
- 🟡 **30–50%** — средняя
- 🟠 **15–30%** — низкая (ответ дан, но может быть неточным)
- ❌ **< 15%** — отказ от ответа

## Ограничения и дальнейшее развитие

**Текущие ограничения:**
- Работает только с русским языком
- Не учитывает контекст диалога (каждый вопрос обрабатывается независимо)
- При одинаково низком сходстве со всеми вопросами может дать нерелевантный ответ

**Что можно улучшить (идеи для доклада):**
- Sentence-Transformers (multilingual-e5-base) — эмбеддинги вместо TF-IDF, accuracy ~95%+
- Добавить логирование вопросов без ответа → расширять базу
- Мультиязычность
- Интеграция с Moodle API для получения реальных дедлайнов
